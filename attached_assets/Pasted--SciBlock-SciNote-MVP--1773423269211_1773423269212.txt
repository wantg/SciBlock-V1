这不是普通详情页，而是一个围绕“本体信息传承 + 当次实验记录 + 后续扩展工具栏”的三栏工作台。

十、本体信息传承机制（必须按这个 mock）
这个逻辑很重要，前端假数据必须按下面来设计。
版本定义
0号本体：初始化阶段系统自动生成的初始本体
用户在初始化流程中修改并确认后，生成 1号本体
1号本体 直接传递给第一次实验记录
第一次实验记录修改确认后，生成 2号本体
第二次实验记录继承 2号本体
依次往下传承
核心原则
每条实验记录创建时，左侧 5 个本体模块默认都不是空白，而是来自上一版已确认本体信息。
也就是说，标题 AI 功能不是面对空白页面工作，而是：
基于当前实验记录中已继承的 5 个本体模块内容
生成标题
生成实验目的
判断哪些模块与当前实验目的相关，并高亮这些模块
十一、前端 mock 数据结构建议
请不要写死成杂乱对象，按清晰结构组织。
至少准备以下数据模型：
1. OntologyVersion
type OntologyModuleKey =
  | "system"
  | "preparation"
  | "operation"
  | "measurement"
  | "data";

type OntologyModule = {
  key: OntologyModuleKey;
  title: string;
  content: string;
  status: "inherited" | "editing" | "confirmed";
  isHighlighted: boolean;
  updatedAt: string;
};

type OntologyVersion = {
  id: string;
  versionNumber: number;
  parentVersionId: string | null;
  source: "initial_generated" | "initial_confirmed" | "experiment_confirmed";
  modules: OntologyModule[];
  confirmedAt?: string;
};
2. ExperimentRecord
type ExperimentStatus = "探索中" | "可复现" | "失败" | "已验证";

type ExperimentRecord = {
  id: string;
  sciNoteId: string;
  title: string;
  purposeInput?: string;
  experimentStatus: ExperimentStatus;
  experimentCode: string;
  tags: string[];
  inheritedOntologyVersionId: string;
  currentModules: OntologyModule[];
  editorContent: string;
  createdAt: string;
};
3. AI mock 返回结构
type PurposeAssistResult = {
  generatedTitle: string;
  purposeDraft: string;
  highlightedModuleKeys: OntologyModuleKey[];
};
